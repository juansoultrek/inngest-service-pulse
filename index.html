Inngest Event Model

Simple, production-like event flow for API and webhook checks.

  • pulse/all.requested starts the global check flow from the UI.
  • pulse/service.requested runs one normalized check (API or webhook URL).
  • pulse/service.checked stores status, latency, and failure metadata.
  • pulse/incident.changed opens or resolves incidents after repeated failures.
